# 개인정보 처리방침 (Privacy Policy)

**확장 프로그램명:** URL Redirect
**최종 수정일:** 2026-05-21

URL Redirect 확장 프로그램(이하 "본 확장")은 사용자의 개인정보를 수집·저장·전송하지 않습니다. 본 문서는 본 확장이 어떤 데이터를 다루는지, 그리고 다루지 않는지를 명확히 하기 위해 작성되었습니다.

## 1. 수집하는 개인정보

**없음.** 본 확장은 다음 어느 정보도 수집하지 않습니다.

- 개인 식별 정보 (이름, 이메일, 전화번호, 주소 등)
- 인증 정보 (비밀번호, 보안 토큰, 결제 정보 등)
- 건강·금융·통신 정보
- 위치 정보
- 웹 검색 기록, 방문 기록, 마우스/키보드 입력 등 사용자 활동
- 기기 식별자 또는 사용 통계

## 2. 본 확장이 다루는 데이터

본 확장은 동작에 필요한 다음 데이터만을 **사용자의 로컬 브라우저 내부**에서 다룹니다.

| 데이터 | 저장 위치 | 용도 |
|---|---|---|
| 사용자가 직접 입력한 리다이렉트 규칙 (패턴, 이동할 URL, 매칭 방식, 동작 방식, 이름) | `chrome.storage.sync` | 규칙에 따른 URL 리다이렉트/새 탭 열기 동작 수행 |
| 규칙 적용 활성/비활성 토글 상태 | `chrome.storage.sync` | 사용자가 일시적으로 모든 규칙을 끄거나 켜기 위함 |
| 현재 열려 있는 탭의 URL | 메모리 (저장 안 됨) | 사용자가 정의한 규칙과의 매칭 평가에 일시적으로 사용 |

`chrome.storage.sync`는 Chrome 브라우저가 제공하는 저장 영역으로, **사용자가 로그인한 Google 계정의 동기화 기능을 통해 동일 계정의 다른 기기 간에만 동기화됩니다.** 이 동기화는 Chrome이 직접 처리하며, 본 확장은 동기화 과정에 관여하지 않습니다.

## 3. 외부 전송 여부

**본 확장은 어떠한 외부 서버, 분석 도구, 광고 네트워크, 제3자 API와도 통신하지 않습니다.**

- 외부 HTTP/HTTPS 요청을 보내지 않습니다.
- 원격 코드를 로드하지 않습니다 (Chrome Manifest V3의 원격 코드 금지 정책을 준수).
- 사용량 통계나 오류 보고를 외부로 전송하지 않습니다.

## 4. 권한 사용 이유

| 권한 | 사용 이유 |
|---|---|
| `tabs` | 탭 URL 변경 감지(`tabs.onUpdated`), 매칭 시 동일 탭 URL 변경(`tabs.update`) 또는 새 탭 생성(`tabs.create`)을 수행하기 위해 사용합니다. |
| `storage` | 사용자가 정의한 규칙과 활성화 상태를 `chrome.storage.sync`에 저장하기 위해 사용합니다. |
| `host_permissions: <all_urls>` | 사용자가 임의의 도메인을 규칙으로 등록할 수 있어 호스트를 사전에 제한할 수 없습니다. 실제 동작은 사용자가 등록한 규칙에 매칭되는 URL에 한해서만 발생하며, 매칭되지 않는 페이지의 내용에는 접근하지 않습니다. |

## 5. 제3자 제공

본 확장은 어떠한 데이터도 제3자에게 판매·공유·제공하지 않습니다. 수집하는 데이터 자체가 없습니다.

## 6. 데이터 삭제

본 확장이 저장한 데이터(규칙 및 설정)는 다음 방법으로 모두 삭제할 수 있습니다.

- **개별 삭제:** 옵션 페이지에서 규칙을 하나씩 삭제합니다.
- **전체 삭제:** Chrome의 `chrome://extensions` 페이지에서 본 확장을 제거하면 `chrome.storage.sync`에 저장된 본 확장의 모든 데이터도 함께 삭제됩니다.

## 7. 정책 변경

본 정책은 확장 기능 업데이트에 따라 변경될 수 있으며, 변경 사항은 본 문서의 "최종 수정일"과 함께 갱신됩니다.

## 8. 문의

본 정책 또는 본 확장 동작에 관한 문의는 Chrome 웹 스토어의 지원 탭을 통해 남겨 주시기 바랍니다.
